An Android app hides its flag behind anti-tooling checks and a time + location gate implemented in a native library (libvault.so). I used Frida to:

  1. Defang process exits
  2. Bypass Java-side checks
  3. Spoof GPS
  4. Spoof time only at Java level so it survives native side effects

With both gates satisfied, the native code XORs two buffers and returns the flag.

1) Recon

Running the APK shows a single button: UNLOCK THE MYTH. Pressing it prints messages like "The gate recognizes your presence..." or nothing, suggesting environmental checks.

Manifest highlights

  • android:debuggable="true"
  • Needs fine location
  • Loads libvault.so via System.loadLibrary("vault")

MainActivity (Kotlin/Java)

Calls a native method:

native String getFlag(double lat, double lon, long currentTime);

Before that it runs three checks:

  • checkDebugger() ‚Üí Debug.isDebuggerConnected()
  • checkEmulator() ‚Üí looks in /system/build.prop and special emulator files
  • checkFrida() ‚Üí scans /data/local/tmp + /proc/*/cmdline + tries TCP 127.0.0.1:27042

It calls getFlag(latitude, longitude, 2000 + System.currentTimeMillis()). (That +2000 ms matters later.)

So even the Java side is anti-tamper aware, but the secret lives in native code.

2) Static triage of libvault.so

Disassembling Java_asc_wargames_myth_MainActivity_getFlag shows a linear sequence of guard functions:

sub_1270  ‚Üí looks for "frida" in /data/local/tmp
sub_1310  ‚Üí walks /proc/*/cmdline, looks for "frida"
sub_16A0  ‚Üí connect("127.0.0.1", 27042)     // frida-server
sub_1750  ‚Üí emulator heuristics (build.prop + special files)
sub_1940  ‚Üí root heuristics (su paths etc.)

On failure paths the function logs via __android_log_print and then calls _exit(1).

If guards pass, it evaluates two gates:

Time gate

In the function's middle there's an immediate constant 0x688DE1A0:

secs = ( (currentTimeMillis + 0x1F4) / 1000 )  // +500 ms
pass if |secs - 0x688DE1A0| ‚â§ 30
  • 0x688DE1A0 = 1754128800 (epoch seconds)
  • The app had already added +2000 ms in Java before passing the time into native

Location gate

Compares the given coordinates to two embedded doubles. From the app code these are:

SACRED_LAT = 30.043801
SACRED_LON = 31.334688

If only time passes ‚Üí returns "TIME".
If only location passes ‚Üí returns "LOCATION".
If both pass ‚Üí it builds the real flag.

Flag construction (sub_19E0)

A simple XOR loop:

    out[i] = A[i] ^ B[i]     // two arrays in .text at 0xC10 and 0xC50
out[0x36] = 0
return NewStringUTF(out)     // via JNIEnv vtable call at offset 0x538

3) First attempts & why they failed

Initially I tried to hook clock_gettime/gettimeofday in libc to spoof time. On this x86 emulator those hooks touched RenderThread/linker/vdso trampolines and occasionally crashed with SIGSEGV in __kernel_vsyscall.

Lesson learned: Don't blanket-patch libc time on Android UI processes unless you must.

4) Plan of attack

Keep the process alive

Replace abort, exit, and _exit with no-ops that just log.

Bypass Java-side checks

Hook MainActivity.checkDebugger/Emulator/Frida() to return false.

Spoof GPS at the Java layer

Return a fabricated Location and hook Location.getLatitude/Longitude() to ensure consistency: 30.043801, 31.334688.

Spoof time only at Java level

  • Read the target epoch from native (scan for 0x688DE1A0) and log it
  • Hook System.currentTimeMillis() to return a value that lands within ¬±30 s after both Java's +2000 ms and native's +500 ms adjustments

A safe pick:

spoofMs = (targetSec + 5) * 1000

‚Üí Java adds 2000 ‚Üí targetSec + 7 seconds
‚Üí Native adds 500ms and divides ‚Üí still ‚âà targetSec + 7
‚Üí |diff| = 7 ‚â§ 30 ‚úÖ

Observe getFlag()

Print the returned jstring.

This approach keeps hooks minimal, avoids fragile inline patches, and works across threads.

Defang process exits

  Interceptor.replace(addr, new NativeCallback((...args) => {
    console.log(`[üõë] ${sym} blocked`);

Bypass Java checks

Java.perform(() => {
  const Main = Java.use('asc.wargames.myth.MainActivity');
  Main.checkDebugger.implementation = () => false;
  Main.checkEmulator.implementation = () => false;
  Main.checkFrida.implementation = () => false;
});

Spoof time (safe)

Java.perform(() => {
  const System = Java.use('java.lang.System');
  const targetSec = 0x688DE1A0;                 // audited from native
  System.currentTimeMillis.implementation = () => (targetSec + 5) * 1000;
});

Spoof GPS

Java.perform(() => {
  const Location = Java.use('android.location.Location');
  const LM = Java.use('android.location.LocationManager');

  // Always hand back a "gps" location with sacred coords
  LM.getLastKnownLocation.implementation = function (provider) {
    const loc = Location.$new(provider || 'gps');
    loc.setLatitude(30.043801);
    loc.setLongitude(31.334688);
    return loc;
  };

  // Belt & suspenders if the instance is read later
  Location.getLatitude.implementation = function () { return 30.043801; };
  Location.getLongitude.implementation = function () { return 31.334688; };
});

8) Takeaways

  • When both Java and native implement checks, intercept where the fewest side effects live (often Java for time & location)
  • If a binary prefers _exit(1), you can often win by neutralizing termination rather than patching every branch
  • XOR-built flags are common; scanning for two nearby constant arrays and a small XOR loop is an instant tell
